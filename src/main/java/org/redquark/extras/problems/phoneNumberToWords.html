Phone Number → Words
Map digits to English words using classic phone keypad mapping.
String
Hash Table
Easy
Given the standard mapping from English letters to digits on a phone keypad (2 → abc, 3 → def, 4 → ghi, 5 → jkl, 6 → mno, 7 → pqrs, 8 → tuv, 9 → wxyz), return all words from a provided dictionary (KNOWN_WORDS) that exactly map to a given n-digit phone number. Words must match the phone number length exactly and mapping is case-insensitive.
Function signature (Java)
public static List<String> wordsFromNumber(String phoneNumber, List<String> knownWords)
Examples
Input: KNOWN_WORDS = ['careers','linkedin','hiring','interview','linkedgo']
phoneNumber = 2273377
Output: ['careers']
Input: KNOWN_WORDS = ['careers','linkedin','hiring','interview','linkedgo']
phoneNumber = 54653346
Output: ['linkedin','linkedgo']
Constraints
n = phoneNumber.length() ≤ 10⁵ across tests; words contain only English letters a–z or A–Z. If you have many queries, precompute a mapping from digit-string → list of words for O(1) queries after preprocessing.
Notes
Digits 0 and 1 have no letter mapping and therefore will not match any purely alphabetic word. The solution is case-insensitive.
