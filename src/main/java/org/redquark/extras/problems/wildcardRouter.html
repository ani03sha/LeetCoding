Wildcard Route Matching

Problem Description

Design a system to register routes and support wildcard path matching. Each route consists of a string path such as "/a/b/c" and is associated with a value. When querying the system, the caller can use a wildcard (*) to match exactly one path segment.

Function Signatures
  • void addRoute(String path, String value) – Registers a new path with an associated value.
  • String getRoute(String query) – Returns the value of a matching path. Supports * as a wildcard for one path segment.
Example
AddRoute("/a/b/c", "value1");
getRoute("/a/*/c"); // returns "value1"

AddRoute("/a", "test");
getRoute("/a"); // returns "test"
Notes
  • The wildcard * matches exactly one path segment.
  • If multiple paths match, the system may return any one valid result (implementations may differ).
  • Paths are case-sensitive and should be normalized (no trailing slash).
  • Assume all inputs are valid and well-formed.
