Tree N-ary BFS Preorder

Extras 22. Compact N‑ary Tree Builder

Description: You are given the root of a tree where each node may have any number of children. Implement a function compact(root, N) that rebuilds the tree so that each node has either exactly N children or is a leaf. At most one node may have between 1 and N-1 children (the exception node).

Ancestor relationships must not be inverted: if node A was an ancestor of node B in the original tree, then in the resulting tree B must not become an ancestor of A. Nodes may be rearranged otherwise.

Approach

  1. Flatten the tree in preorder (ancestor before descendants) into a list of nodes.
  2. Rebuild the tree level-by-level, giving each node up to N children, until all nodes are assigned.

Example

Input: Tree rooted at A, N = 2
Output: A compacted tree where every node has 2 or 0 children, with at most one having 1 child.

Function Signature

Node<T> compact(Node<T> root, int N)

Constraints

  • 1 ≤ N
  • Tree size up to environment limits.
  • Node values can be of any type.
