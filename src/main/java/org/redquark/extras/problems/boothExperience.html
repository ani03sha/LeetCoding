Maximize Booth Experience Factor
String • Prefix Sum • Range Query
Array
Prefix Sum
Easy
Problem
You are given m booths arranged in a line. Each booth displays either a robot (0) or a drone (1), represented by a binary string booths of length m. You are also given n queries. The k-th query specifies a 1-based inclusive range [i, j], indicating that the k-th group visits exactly the booths from position i to j.

For a range [i, j], define:
ones = number of '1' in booths[i..j]
zeros = (j - i + 1) - ones
experience(i, j) = ones × zeros
For each query, compute experience(i, j).
Function Signature
public long[] experienceFactors(String booths, int[][] queries)
Input/Output Format
Input:
  • booths: binary string of length m consisting only of characters '0' and '1'.
  • queries: array of length n, where each element is [i, j] (1-based inclusive).
Output:
  • Array ans of length n, where ans[k] = experience(i_k, j_k).
Examples
Example 1

Input:
booths = "1100101"
queries = [[2,6],[1,7],[3,5]]
Output:
[6, 12, 2]
Explanation:
For [2,6]: substring "10010" has ones=2, zeros=3 → 2×3=6.
For [1,7]: substring "1100101" has ones=4, zeros=3 → 4×3=12.
For [3,5]: substring "001" has ones=1, zeros=2 → 1×2=2.
Example 2

Input:
booths = "00000"
queries = [[1,5],[2,4]]
Output:
[0, 0]
All characters are '0', so ones=0 in any range and the product is always 0.
Example 3

Input:
booths = "1"
queries = [[1,1]]
Output:
[0]
Single booth with '1': ones=1, zeros=0 → product 0.
Constraints
[Inference] Typical limits for an efficient prefix-sum solution:
  • [Inference] 1 ≤ m ≤ 2×105
  • [Inference] 1 ≤ n ≤ 2×105
  • [Inference] Indices in queries are 1-based and satisfy 1 ≤ i ≤ j ≤ m.
  • Time goal: O(m + n) using prefix sums of '1' counts.
Approach
Build a prefix array pref where pref[k] is the number of '1' in booths[0..k-1]. For each query [i, j]:
ones = pref[j] - pref[i-1]
len = j - i + 1
zeros = len - ones
ans = ones × zeros
This answers each query in O(1) after O(m) preprocessing.
Clarification: This problem interprets each group’s range [i, j] as fixed. You do not choose a subrange inside [i, j]; you simply compute the product for that exact range.
