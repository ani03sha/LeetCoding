Design Concurrent LRU Cache

Design and implement a thread-safe LRU (Least Recently Used) Cache. Your implementation should support concurrent get and put operations with proper synchronization to avoid race conditions.

Implement the class:

ConcurrentLRUCache<K, V> {
  ConcurrentLRUCache(int capacity)
  V get(K key)
  void put(K key, V value)
}

Constraints:

  • The cache must support O(1) average time complexity for get and put operations.
  • Must handle concurrent access from multiple threads safely.
  • Evict the least recently used entry when the cache exceeds capacity.

Example:


ConcurrentLRUCache<Integer, Integer> cache = new ConcurrentLRUCache<>(2);
cache.put(1, 1);
cache.put(2, 2);
cache.get(1);       // returns 1
cache.put(3, 3);    // evicts key 2
cache.get(2);       // returns null (not found)
